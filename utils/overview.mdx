# Utils overview

**fluide-utils** is the shared utilities microservice for the Fluide Suite. Product APIs delegate cross-cutting work here so each domain service stays focused on its core domain.

## Base URL

All utils endpoints are under:

- `/api/v1/app/*`

Authenticate using developer keys described in [authentication](/getting-started/authentication).

## Responsibilities

| Module | Purpose |
| --- | --- |
| Notifications | In-app and templated notifications for HR, payroll, and engagement flows |
| File Management | Organization-scoped uploads and downloads |
| Documents | PDF and spreadsheet generation (payslips, invoices, financial reports) |
| Document Jobs | Async generation with job status polling |
| Email | Transactional email delivery via Kafka (internal; not a public REST surface) |

## Who calls Utils

- **Payroll** — payslip PDF generation after a run completes
- **Books** — invoice PDFs and financial statement exports
- **HR / Admin** — file uploads and user notifications
- **Gateway** — routes `/api/v1/app/*` with `utils:read` / `utils:write` scopes

## Operations

| Endpoint | Purpose |
| --- | --- |
| `GET /api/v1/app` | Service root |
| `GET /api/v1/app/metrics` | Prometheus metrics |
